# CLAUDE.md

## Release Engineering Variant

- Treat rollout, rollback, and monitoring as part of the change, not follow-up trivia.
- Call out config changes, migrations, feature flags, and operator steps explicitly.
- Prefer release notes that distinguish user-visible impact from operator impact.
- Surface missing verification before release instead of assuming post-deploy fixes.
- End with rollback confidence, monitoring plan, and residual operational risk.

# CLAUDE.md

## Internal Tooling Defaults

- Prefer the smallest relevant admin, auth, or policy test path before broad test runs.
- Call out permission boundaries and destructive-action safeguards when internal screens or endpoints change.
- Treat operator-facing wording, confirmation steps, and audit visibility as part of correctness.
- If the change affects privileged workflows, summarize residual operational risk before calling the task complete.

# Agent Catalog

These are the shared project subagents included in the starter.

Use them as-is, or treat them as examples for your own team-specific roles.

## `code-reviewer`

Use when you want:

- findings-first review of the current diff
- regression and test-risk checks
- a second pass before merge

## `debugger`

Use when you want:

- root-cause analysis
- failure isolation
- a small, evidence-driven fix

## `docs-writer`

Use when you want:

- concise docs updates
- release notes
- changelog or handoff writing

## `release-manager`

Use when you want:

- rollout notes
- release checklists
- risk and follow-up tracking

## `security-reviewer`

Use when you want:

- auth and authorization review
- secret-handling checks
- trust-boundary or shell-safety review

## `test-runner`

Use when you want:

- the smallest relevant test command
- reproduction of failures
- minimal fixes with verification
